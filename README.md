# monocopy

Static, copy-safe text page for factory embroidery monograms.

Single-file `index.html`. Zero dependencies. Zero network requests after load. No frameworks, no build step.

---

## What this is

A "ghost page" hosted on Cloudflare Pages at **https://monocopy.pages.dev/**. It exists to solve one problem:

> Factory embroidery workers in China need to paste monogram text (e.g. `CO'C`, `FT005`) from an order bill into their embroidery machine without the PDF-viewer copy-paste heuristic injecting stray spaces.

The page reads a base64url-encoded JSON payload from the `?t=` query parameter, renders one card per monogram, and offers a single-tap "复制 / Copy" button with a three-layer clipboard fallback. Mandarin-primary. WeChat Android WebView compatible.

## Who scans this

Factory embroidery workers. The QR code is printed in the bottom-left corner of the factory-scoped order bill PDF (generated by the MTM Laravel app). Workers scan the QR with their phone camera → page opens in a browser → tap Copy → paste into the embroidery machine input.

## How the payload gets here

1. Laravel app builds a JSON array of monograms from the order's designs:
   ```json
   [
     {"placement": "内袋", "text": "CO'C"},
     {"placement": "袖口", "text": "FT005"}
   ]
   ```
2. JSON is encoded with `JSON_UNESCAPED_UNICODE` (so CJK characters stay as UTF-8 bytes, not `\uXXXX` escapes).
3. The result is base64url-encoded — standard base64 with `+/` replaced by `-_` and `=` padding stripped. This keeps the string safe inside a URL without further percent-encoding.
4. The resulting URL `https://monocopy.pages.dev/?t=<base64url>` is rendered as a QR code in the PDF.

The Laravel helpers that do this live in:
- `app/Support/MonogramPayloadBuilder.php`
- `app/Http/Controllers/Admin/PDFController.php`

Base URL is configurable via `MONOGRAM_COPY_URL` (in `config/app.php`) so staging/prod can point elsewhere if needed.

## Query-param format

```
https://monocopy.pages.dev/?t=<base64url-of-JSON-array>
```

- Payload: a JSON array of `{placement: string, text: string}` objects
- `placement` may be an empty string — the text is still rendered (metadata failures never swallow the monogram)
- `text` is required for each entry; empty entries are skipped defensively
- Encoding: `base64(JSON.stringify(payload, unescapeUnicode=true)).replace('+', '-').replace('/', '_').rstrip('=')`

## Debugging a broken QR link

1. Decode the `?t=` value locally:
   ```bash
   node -e "
     const t = process.argv[1];
     const b64 = t.replace(/-/g,'+').replace(/_/g,'/') + '==='.slice((t.length + 3) % 4);
     console.log(JSON.parse(Buffer.from(b64, 'base64').toString('utf8')));
   " '<paste-t-value-here>'
   ```
2. Or in a browser console on the page:
   ```js
   const t = new URLSearchParams(location.search).get('t');
   const b64 = t.replace(/-/g,'+').replace(/_/g,'/') + '==='.slice((t.length + 3) % 4);
   JSON.parse(decodeURIComponent(escape(atob(b64))));
   ```
3. Test the page directly with a known-good payload:
   - No `?t=` → gentle empty state: "未找到刺绣文字数据"
   - `?t=garbage` → gentle error state: "二维码数据无法读取"
   - Valid `?t=...` → one card per monogram

## States

| URL | Result |
|---|---|
| `/` (no `?t=`) | Empty state: "未找到刺绣文字数据 · No monogram data" |
| `/?t=<garbage>` | Error state: "二维码数据无法读取 · Unable to read QR data" |
| `/?t=<base64url>` with empty array | Error state (same as bad payload) |
| `/?t=<base64url>` with valid array | One card per monogram, Copy button each |

## Clipboard — three-layer fallback

1. **Layer 1** — `navigator.clipboard.writeText(text)` (modern browsers, HTTPS)
2. **Layer 2** — hidden `<textarea>` + `document.execCommand('copy')` (covers older Android WebView, most WeChat contexts)
3. **Layer 3** — `window.getSelection()` range-selects the text block, button label changes to `选择文本并长按复制`, the monospace block gets a pulse highlight. Guarantees the user can always copy manually.

Success state: `✓ 已复制 / Copied` — reverts after 2 seconds.

## Deployment

- **Host:** Cloudflare Pages (free tier, `*.pages.dev`)
- **Auto-deploy:** push to `main` in this repo → Cloudflare Pages redeploys within ~1 minute
- **Build command:** _(none — static file)_
- **Output directory:** `/`
- **Env vars:** none

The Cloudflare Pages project must be connected to this GitHub repo once via the Cloudflare dashboard (Workers & Pages → Create → Pages → Connect to Git → select `RJmtm/monocopy`). That's a one-time USER ACTION, not part of the repo.

## Constraints (do not violate)

- **No external fonts** (no Google Fonts, no CDN). System CJK stack only: `-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "WenQuanYi Micro Hei", "Helvetica Neue", Helvetica, Arial, sans-serif`
- **No network requests** after initial HTML load. Verify in DevTools → Network tab (should be exactly 1 request: the HTML itself)
- **No analytics, no telemetry.** This is a ghost page — it exists to render text and disappear.
- **No CSS `:has()`, no container queries, no CSS nesting.** Target floor is ~2020 Chrome (WeChat Android WebView).
- **No build step.** Single-file static HTML with inline CSS + inline JS. Open `index.html` directly to verify.
- **No console logging.** The page is silent.
